<Note>
  Ozel Arama Ici Araclar, yapay zeka asistaninizin aramalar sirasinda dis
  sistemlerle etkilesim kurmasini saglar. Stok kontrolu, musteri verisi
  dogrulama veya gercek zamanli bilgi alma gibi islemlerle yapay zekanizi daha
  guclu ve baglantili hale getirir.
</Note>

## Genel Bakis

Ozel Arama Ici Araclar, yapay zeka asistaninizin sunlari yapmasini saglar:

- Konusmalar sirasinda gercek zamanli API cagrilari yapma
- Sistemlerinizden veri cekme veya veri gonderme
- Canli verilere dayanarak bilinçli kararlar alma
- Arayanlara dogru ve guncel bilgi saglama

<Card title="Temel Avantajlar" icon="star">
  - Kodlama gerektirmez - sadece API ucu noktasini ve parametreleri yapilandirin - Yapay zeka
  araclari ne zaman ve nasil kullanacagini otomatik olarak bilir - Aramalar
  sirasinda gercek zamanli veri erisimi - Mevcut sistemlerinizle sorunsuz entegrasyon
</Card>

## Aracinizi Kurma

### 1. Temel Yapilandirma

<Frame>
  <img src="/resources/mid-call.png" alt="Ana Arac Yapilandirmasi" />
</Frame>

<Steps>
  <Step title="Arac Olusturmaya Erisin">
    **Arama ici araclar** bolumune gidin ve **Arama Ici Arac Olustur** dügmesine tiklayin
  </Step>

  <Step title="Ana Ayarlari Yapilandirin">
    Temel bilgileri doldurun:

    - **Ad**: Kucuk harf ve alt cizgi kullanin (orn. `siparis_durumu_kontrol`)
    - **Aciklama**: Yapay zekanin bu araci ne zaman ve nasil kullanmasi gerektigini aciklayin
    - **Uç Nokta**: API URL'niz (orn. `https://api.sirketiniz.com/siparisler`)
    - **Zaman Asimi**: Yanitlar icin ne kadar beklenmesi gerektigi (saniye cinsinden)
    - **Yontem**: GET, POST, PUT, PATCH veya DELETE secin

  </Step>

  <Step title="Basliklar Ekleyin">
    Ihtiyac duyabileceginiz yaygin basliklar:
    ```yaml
    Content-Type: application/json
    Authorization: Bearer sizin_tokeniniz
    ```
  </Step>
</Steps>

### 2. Degisken Yapilandirmasi

<Frame>
  <img src="/resources/mid-call-param.png" alt="Degisken Yapilandirmasi" />
</Frame>

<Steps>
  <Step title="Parametreleri Tanimlayin">
    Bunlar, yapay zekanizin arama sirasinda toplayacagi bilgi parcalaridir:
    
    ```yaml
    Ad: siparis_numarasi
    Tur: string
    Aciklama: "Musteriden alinan 10 haneli siparis numarasi"
    ```
  </Step>

  <Step title="Dogrulama Kurallarini Belirleyin">
    Aciklamaya format gereksinimlerini ekleyin:
    
    ```yaml
    "gg/aa/yyyy formatinda tarih"
    "Bosluksuz telefon numarasi"
    "Onay icin e-posta adresi"
    ```
  </Step>
</Steps>

## Dinamik Uç Noktalar

<Warning>
  Uç nokta URL'nizde degiskenler kullanirken, bunlari suslu parantez icine
  aldiginizdan ve tam parametre adini kullandiginizdan emin olun.
</Warning>

Degiskenler kullanarak uç noktalarinizi dinamik hale getirebilirsiniz:

```yaml
Temel URL:
https://api.ornek.com/siparisler/durum

Degiskenli:
https://api.ornek.com/siparisler/{siparis_id}/durum
```

Yapay zeka, konusma sirasinda toplanan gercek degerle `{siparis_id}` ifadesini otomatik olarak degistirecektir.

## Ileri Duzey Ozellik: Otomasyon Platformu Entegrasyonu

<Note>
  Daha karmasik mantik mi gerekiyor? Gelismis is akislari icin aracinizi
  otomasyon platformuna baglayin.
</Note>

Gelismis entegrasyonlar ve karmasik is akislari icin, ozel araclari [Otomasyon Platformumuz](/tr/automation-platform/introduction) ile birlestirebilirsiniz.

Guclu entegrasyonlar olusturmanin yolu:

1. Otomasyon platformunda:

   - Yeni bir akis olusturun
   - Bir webhook tetikleyicisi ekleyin
   - Mantiginizi olusturun (API cagrilari, veri donusumu vb.)
   - "Yanit Dondur" ile bitirin

2. Ozel aracinizda:
   - Webhook URL'sini uç nokta olarak kullanin
   - Sonuna `/sync` ekleyin
   - Ornek: `https://automation.whattalk.ai/webhook/abc123/sync`

Bu size sunlari saglar:

- API cagrilarindan once/sonra veri donusumu
- Sirali olarak birden fazla API cagrisi yapma
- Karmasik is mantigi uygulama
- Hatalari duzgun bir sekilde yonetme

## Yapay Zekanizi Yapilandirma

<Info>
  Yapay zekanin ozel araclarinizi etkili bir sekilde kullanabilmesi icin sistem
  isteminde net talimatlara ihtiyaci vardir.
</Info>

Ornek istem bolumu:

```yaml
siparis_kontrol aracini ne zaman kullanmali:
1. Musteri siparis durumunu sorar
2. Takip veya teslimattan bahseder
3. Paketinin nerede oldugunu bilmek ister

Nasil kullanilir:
1. Verilmediyse siparis numarasini sorun
2. Formati dogrulayin (SIP-XXXXX)
3. Durumu sorgulamak icin araci kullanin
4. Sonuclari basit bir dille aciklayin
```

<Tip>
  Yapay zekanin tum senaryolari sorunsuz bir sekilde yonettiginden emin olmak
  icin araclarinizi cesitli konusma akislariyla test edin. Yayina almadan once
  basit test aramalariyla baslatin.
</Tip>
