---
title: "Şablonları Getir"
api: "GET /user/whatsapp/senders/{senderId}/templates"
description: "Belirli bir gönderen için WhatsApp mesaj şablonlarını listeleyin"
---

Bu endpoint, belirli bir WhatsApp gönderenine ait tüm mesaj şablonlarını döndürür. Şablonlar, konuşmaları başlatmak veya 24 saatlik mesajlaşma penceresi dışında kullanıcılara mesaj göndermek için gereklidir.

### Yol Parametreleri

<ParamField path="senderId" type="integer" required>
  WhatsApp göndereninin kimliği ([Gönderenleri Getir](/api-reference/whatsapp/get-senders) endpoint'inden elde edilir)
</ParamField>

### Sorgu Parametreleri

<ParamField query="status" type="string" optional>
  Şablonları onay durumuna göre filtreleyin. Varsayılan: `approved`. Durumdan bağımsız olarak tüm şablonları döndürmek için `all` kullanın.
</ParamField>

### Yanıt Alanları

<ResponseField name="data" type="array">
  <Expandable title="Şablon nesnesi özellikleri">
    <ResponseField name="id" type="integer">
      Şablonun benzersiz tanımlayıcısı. Şablon mesajları gönderirken bunu kullanın.
    </ResponseField>

    <ResponseField name="name" type="string">
      Meta'ya kayıtlı şablon adı (ör. `order_confirmation`, `appointment_reminder`)
    </ResponseField>

    <ResponseField name="language" type="string">
      Şablon dil kodu (ör. `en`, `es`, `pt_BR`)
    </ResponseField>

    <ResponseField name="category" type="string">
      Şablon kategorisi: `marketing`, `utility` veya `authentication`
    </ResponseField>

    <ResponseField name="status" type="string">
      Onay durumu: `approved`, `pending` veya `rejected`
    </ResponseField>

    <ResponseField name="body_text" type="string">
      Şablon gövde metni, değişken yer tutucuları `{{1}}`, `{{2}}` vb. olarak gösterilir.
    </ResponseField>

    <ResponseField name="variables" type="array">
      Şablon için tanımlanan değişken adları listesi. Şablonda değişken yoksa boş dizi.
    </ResponseField>

    <ResponseField name="has_variables" type="boolean">
      Bu şablonun gönderirken değişken sağlanmasını gerektirip gerektirmediği
    </ResponseField>
  </Expandable>
</ResponseField>

### Hata Yanıtları

<ResponseField name="404 Not Found">
  <Expandable title="Hata Yanıtı">
    <ResponseField name="success" type="boolean">
      `false`
    </ResponseField>
    <ResponseField name="error" type="string">
      `Sender not found`
    </ResponseField>
    <ResponseField name="error_code" type="string">
      `SENDER_NOT_FOUND`
    </ResponseField>
  </Expandable>
</ResponseField>

<RequestExample>

```bash cURL
curl -X GET "https://app.whattalk.ai/api/user/whatsapp/senders/12/templates" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```bash Tüm şablonlar (bekleyen/reddedilen dahil)
curl -X GET "https://app.whattalk.ai/api/user/whatsapp/senders/12/templates?status=all" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```javascript JavaScript
const senderId = 12;

const response = await fetch(
  `https://app.whattalk.ai/api/user/whatsapp/senders/${senderId}/templates`,
  {
    headers: {
      'Authorization': 'Bearer YOUR_API_KEY'
    }
  }
);

const data = await response.json();
console.log(data.data); // Array of templates
```

```python Python
import requests

sender_id = 12

response = requests.get(
    f'https://app.whattalk.ai/api/user/whatsapp/senders/{sender_id}/templates',
    headers={'Authorization': 'Bearer YOUR_API_KEY'}
)

templates = response.json()['data']
for template in templates:
    print(f"{template['name']} ({template['language']}): {template['body_text']}")
```

### Notlar

- Varsayılan olarak yalnızca `approved` şablonlar döndürülür. `pending` veya `rejected` durumundaki şablonlar mesaj göndermek için kullanılamaz.
- Şablon onay durumu her 4 saatte bir Meta ile otomatik olarak senkronize edilir.
- `body_text` içindeki değişkenler `{{1}}`, `{{2}}` vb. olarak gösterilir. `variables` dizisi her yer tutucu için okunabilir adlar sağlar.
- Bir kullanıcıya ilk kez mesaj gönderirken veya 24 saatlik mesajlaşma penceresi dışında şablonlar gereklidir.
